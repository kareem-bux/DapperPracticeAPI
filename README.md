# DapperPracticeAPI
This repository is a hands-on practice project for building a RESTful API using ASP.NET Core 8 and Dapper. It showcases best practices for lightweight and efficient data access while demonstrating real-world API patterns.

# Key Features

1 CRUD Operations using both raw SQL queries and stored procedures

2 Repository Pattern implementation for clean data access

3 Parameterized Queries to prevent SQL injection

4 Transactions for reliable multi-step database operations

5 Multi-Mapping examples for complex object relationships

6 Clean Architecture Principles to ensure scalable and maintainable code

7 Integration with SQL Server LocalDB for easy setup and testing

Purpose

This project is designed for developers who want to learn and practice Dapper in a realistic API scenario, combining performance, simplicity, and maintainable architecture. It is also suitable as a starting template for building production-ready APIs with Dapper.
